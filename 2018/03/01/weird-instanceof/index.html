怪异的 instanceof

2018-03-01
技术文章 思考总结
阅读次数 0

代码来自 wtfjs - a little code blog about that language we love despite giving us so much to hate

当我们把构造函数的原型设置为 null 时，instanceof 操作符出现了怪异的表现：

调用 o instanceof C 时实际上是调用了 C[@@hasInstance](o)。
在浏览器中 instanceof 操作符等同于 Function.prototype[Symbol.hasInstance] 函数 ，即 a instanceof Test 等同于 Test[Symbol.hasInstance](a)。

而 Function.prototype[Symbol.hasInstance] 内部实际上是调用了一个叫做 OrdinaryHasInstance(C, O) 的内部方法。该方法简单来说有下面这几个步骤：

  • 判断 C 是否可调用，若不可调用则返回 false
  • 判断 O 是否为对象，若不是对象则返回false
  • 获取构造函数 C 的原型 P
    • 判断 P 是否为对象，若不是对象抛出 TypeError 异常
  • 重复执行以下步骤
    • 令 O 为 Object.getPrototypeOf(O)
    • 判断 O 是否为 null（是否到达原型链终点），若是则返回 false
    • 判断 O 是否与 P 相同，若相同则返回 true

所以这就解释了为什么当 Test 的原型被设置为 null 时，抛出了 TypeError 异常。需要注意的是 OrdinaryHasInstance(C, O) 内部方法判断类型使用的不是 typeof 而是内部方法 Type。并且因为 Object.getPrototypeOf(a) === Object.prototype， 所以 a instanceof Object 返回的是 true。
